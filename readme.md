# Access Control for Visitors

## Описание функционала
Плагин ограничивает доступ к материалам сайта для неавторизованных посетителей.

**Возможности:**
- Обрезка контента до первого тега H2
- Обрезка контента по лимиту символов (если H2 нет)
- Сохранение HTML-структуры при обрезке
- Добавление сообщения со ссылками на вход/регистрацию
- Скрытие комментариев и формы комментариев для гостей
- **Удаление элементов с произвольными CSS-классами** (серверное удаление из HTML)
- Настройка URL входа, регистрации, лимита символов и исключаемых классов в админке

## Основные методы классов

**ACV_Frontend:**
- `restrict_content()` — обрезка контента
- `truncate_html()` — обрезка HTML с сохранением тегов
- `get_access_message()` — формирование сообщения доступа
- `disable_comments()` — отключение комментариев
- `hide_comments()` — скрытие комментариев
- `remove_comment_form()` — удаление формы комментариев

**ACV_Admin:**
- `add_admin_menu()` — добавление меню
- `register_settings()` — регистрация настроек
- `render_settings_page()` — вывод страницы настроек
- `render_login_url_field()` — поле URL входа
- `render_register_url_field()` — поле URL регистрации
- `render_char_limit_field()` — поле лимита символов
- `render_exclude_classes_field()` — поле ввода CSS-классов для исключения

**ACV_CSS_Hide:**
- `filter_content()` — фильтр контента для неавторизованных; удаляет элементы с указанными классами
- `remove_elements_by_classes()` — вспомогательный метод, удаляющий узлы DOM по списку классов

## Настройка исключения элементов по классам
1. В админ-панели плагина («Настройки доступа для гостей») в поле «Исключаемые CSS-классы» укажите через запятую классы элементов, которые должны быть полностью удалены из HTML для неавторизованных посетителей (например, `table-of-contents, no-guest, private`).
2. Сохраните настройки.
3. Для гостей указанные элементы перестанут появляться на странице — они удаляются на сервере до отправки клиенту.
